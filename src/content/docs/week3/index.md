# Docker — 한 줄이면 설치, 한 줄이면 삭제

코딩의 세계에 발을 들인 여러분을 환영합니다. 처음 공부를 시작하면 가장 먼저 마주하는 벽은 코드 그 자체보다 코드를 실행하기 위한 환경 설정인 경우가 많습니다. 파이썬을 설치하고, 데이터베이스를 설정하고, 웹 서버를 올리는 과정에서 설정이 꼬여 컴퓨터를 초기화하고 싶었던 경험이 한 번쯤은 있으실 것입니다. 도커는 바로 이러한 번거로움을 해결하기 위해 등장한 도구입니다.

## 1. Docker가 뭔가요?

도커를 이해하는 가장 좋은 방법은 이사를 가는 상황을 떠올려 보는 것입니다. 우리가 살던 집에서 다른 집으로 이사를 갈 때, 침대, 책상, 옷장, 컴퓨터를 하나씩 따로 옮긴다고 가정해 봅시다. 새집에 도착해서 침대 프레임을 다시 조립하고 책상 수평을 맞추는 일은 무척 고된 작업입니다. 심지어 새집의 방 크기가 예전 집과 다르거나 콘센트 위치가 맞지 않으면 가구를 배치하는 과정에서 큰 혼란이 생깁니다.

도커는 이 방 전체를 통째로 커다란 상자에 넣어서 옮기는 기술과 같습니다. 방 안에 있는 가구, 전등, 벽지, 심지어 공기의 온도와 습도까지 그대로 상자에 담습니다. 그리고 새집에 도착하면 그 상자만 열어 놓습니다. 그러면 그 상자 안은 우리가 원래 살던 방과 똑같은 상태가 됩니다. 집의 구조가 어떻든 상자 안의 환경은 변하지 않습니다.

소프트웨어 세계에서도 마찬가지입니다. 우리가 만든 프로그램이 내 컴퓨터에서는 잘 돌아가는데 친구 컴퓨터에서는 작동하지 않는 일이 자주 벌어집니다. 친구 컴퓨터에는 특정 소프트웨어가 설치되어 있지 않거나 버전이 다르기 때문입니다. 도커를 사용하면 프로그램과 그 프로그램이 실행되는 데 필요한 모든 설정, 도구, 라이브러리를 컨테이너라는 상자에 담습니다. 이 상자만 있으면 전 세계 어디서든 똑같은 모습으로 프로그램을 실행할 수 있습니다.

### 내 컴퓨터를 깨끗하게 유지하는 비결

도커를 사용하면 내 컴퓨터 본체에 직접 프로그램을 설치하지 않아도 됩니다. 모든 작업은 컨테이너라는 독립된 공간 안에서 이루어집니다. 덕분에 여러 프로그램을 설치해도 내 컴퓨터의 시스템 설정이 오염되거나 꼬이는 일이 발생하지 않습니다. 프로그램을 지우고 싶을 때는 그저 컨테이너라는 상자만 통째로 버리면 그만입니다. 흔적조차 남지 않습니다.

### 가상머신과는 무엇이 다른가요?

기존에도 가상머신이라는 기술이 있었습니다. 내 컴퓨터 안에 또 다른 가상 컴퓨터를 만드는 방식입니다. 하지만 가상머신은 윈도우나 리눅스 같은 운영체제를 통째로 새로 설치해야 하기 때문에 덩치가 매우 크고 컴퓨터 자원을 많이 차지합니다. 이사를 갈 때 방을 옮기는 게 아니라 아예 집 한 채를 새로 짓는 것과 비슷합니다.

반면 도커는 운영체제의 핵심 부분은 내 컴퓨터와 공유하면서 필요한 프로그램들만 상자에 담습니다. 운영체제를 새로 설치하지 않으므로 훨씬 가볍고 빠르게 작동합니다. 집을 새로 짓는 대신 미리 만들어진 방 하나를 헬리콥터로 옮겨와 설치하는 수준의 속도를 보여줍니다.

## 2. 핵심 용어 3가지

도커를 다룰 때 반드시 알아야 할 세 가지 개념이 있습니다. 요리에 비유해서 설명해 보겠습니다.

### 이미지(Image): 레시피

이미지는 특정 프로그램을 실행하기 위한 모든 정보가 담긴 설명서이자 레시피입니다. 이 안에는 어떤 재료가 들어가야 하는지, 어떤 순서로 조리해야 하는지가 아주 상세히 기록되어 있습니다. 이미지는 변하지 않는 파일입니다. 한 번 만들어진 레시피가 책에 적히면 내용이 바뀌지 않는 것과 같습니다.

### 컨테이너(Container): 실제 요리

레시피를 보고 실제로 만든 요리가 바로 컨테이너입니다. 하나의 이미지(레시피)를 가지고 여러 개의 컨테이너(요리)를 만들어낼 수 있습니다. 김치찌개 레시피가 하나 있다면 식탁 위에 김치찌개를 세 그릇, 네 그릇 차려낼 수 있는 것과 같은 원리입니다. 각 컨테이너는 서로 영향을 주지 않고 독립적으로 존재합니다. 한 그릇에 설탕을 더 넣는다고 해서 다른 그릇의 맛이 변하지 않습니다.

### 레지스트리(Registry): 레시피 책방

전 세계 사람들이 자신이 만든 레시피를 공유하는 도서관이나 책방을 레지스트리라고 부릅니다. 가장 대표적인 곳이 도커 허브입니다. 이곳에 가면 웹 서버를 돌리는 방법, 데이터베이스를 구축하는 방법 등 이미 검증된 수많은 레시피가 저장되어 있습니다. 우리는 그저 원하는 레시피의 이름을 검색해서 내 컴퓨터로 가져오기만 하면 됩니다.

## 3. Docker Desktop 설치하기

도커를 사용하기 위해서는 도커 데스크탑이라는 프로그램을 설치해야 합니다. 운영체제별 설치 방법을 안내합니다.

### Windows 설치 방법

1. 공식 설치 페이지(https://docs.docker.com/desktop/setup/install/windows-install/)에 접속합니다.
2. 설치 파일을 내려받아 실행합니다.
3. 설치 과정에서 나타나는 기본 옵션들을 그대로 두고 진행합니다.
4. 설치가 완료되면 컴퓨터를 재부팅합니다.
5. 바탕화면이나 시작 메뉴에서 도커 데스크탑을 실행합니다. 오른쪽 하단 트레이 아이콘에 귀여운 고래 모양이 나타나면 준비가 된 것입니다.

**설치 전 확인 사항:**
- 윈도우 10 빌드 19045 버전 이상이 필요합니다.
- 64비트 환경이어야 하며 최소 4기가바이트 이상의 메모리를 갖추어야 합니다.
- 컴퓨터 바이오스 설정에서 가상화 기능이 활성화되어 있어야 합니다. 최근 컴퓨터들은 대부분 기본적으로 켜져 있습니다.
- 윈도우용 리눅스 하위 시스템인 WSL 2가 먼저 설치되어 있으면 성능이 더 안정적입니다.

### macOS 설치 방법

1. 공식 설치 페이지(https://docs.docker.com/desktop/setup/install/mac-install/)에 접속합니다.
2. 사용 중인 맥의 프로세서 종류에 맞는 파일을 선택합니다. M1, M2, M3, M4 등 최신 프로세서를 사용한다면 애플 실리콘용 파일을, 과거의 맥이라면 인텔용 파일을 내려받습니다.
3. 내려받은 파일을 열고 고래 아이콘을 응용 프로그램 폴더로 끌어다 놓습니다.
4. 응용 프로그램 폴더에서 도커를 찾아 실행합니다.

**설치 전 확인 사항:**
- 맥OS 11 버전 이상이 필요합니다.
- 최소 4기가바이트 이상의 메모리가 필요합니다.

### 설치가 잘 되었는지 확인하기

터미널이나 명령 프롬프트를 열고 다음 명령어를 입력해 봅니다.

```bash
docker --version
```

도커의 버전 정보가 출력된다면 설치에 성공한 것입니다. 이어서 다음 명령어를 입력해 봅니다.

```bash
docker run hello-world
```

이 명령어는 도커가 잘 작동하는지 테스트하기 위한 가장 작은 프로그램을 실행하는 명령어입니다. 화면에 환영 메시지가 나타난다면 모든 준비가 끝났습니다. 이 과정에서 도커는 레지스트리에서 헬로 월드 이미지를 찾아 다운로드하고 이를 컨테이너로 만들어 실행한 뒤 종료합니다.

## 4. 처음 사용해보기 — 서비스를 한 줄로 설치하기

이제 도커의 진가를 확인해 볼 시간입니다. 복잡한 설치 과정 없이 명령어 한 줄로 강력한 서비스들을 실행해 보겠습니다.

### 예제 1: 웹 서버 실행하기

웹 서버를 내 컴퓨터에 직접 설치하려면 설정 파일을 수정하고 포트를 열어주는 등 복잡한 과정을 거쳐야 합니다. 도커로는 다음과 같이 한 줄이면 충분합니다.

```bash
docker run -d -p 8080:80 --name my-web nginx
```

명령어를 입력한 후 인터넷 브라우저 주소창에 `http://localhost:8080`을 입력해 보세요. 웹 서버가 성공적으로 작동한다는 메시지를 볼 수 있습니다. 사용된 옵션의 의미를 살펴보겠습니다.

- `-d`: 프로그램을 배경에서 실행합니다. 터미널 창을 닫아도 웹 서버는 계속 돌아갑니다.
- `-p 8080:80`: 내 컴퓨터의 8080번 포트와 컨테이너의 80번 포트를 연결합니다. 외부에서 8080번으로 들어오면 컨테이너 안의 웹 서버로 전달해 줍니다.
- `--name my-web`: 이 상자의 이름을 내가 알아보기 좋게 지정합니다.
- `nginx`: 사용할 이미지의 이름입니다. 전 세계에서 가장 많이 쓰이는 웹 서버 레시피입니다.

### 예제 2: 데이터베이스 실행하기

데이터베이스 설치는 코딩 입문자들에게 가장 큰 고난 중 하나입니다. 하지만 도커가 있다면 역시 한 줄로 해결됩니다.

```bash
docker run -d -p 5432:5432 --name my-db -e POSTGRES_PASSWORD=1234 postgres
```

이 명령어 한 줄로 강력한 데이터베이스 엔진인 포스트그레스큐엘이 설치되고 실행됩니다. `-e` 옵션은 환경 변수를 설정하는 것으로 여기서는 데이터베이스의 비밀번호를 설정했습니다.

### 설치한 서비스 삭제하기

도커의 가장 멋진 점은 삭제가 설치만큼이나 명료하다는 것입니다. 방금 만든 웹 서버를 삭제해 보겠습니다.

```bash
docker stop my-web
docker rm my-web
```

먼저 실행 중인 상자를 멈추고 그다음 상자를 버립니다. 이제 내 컴퓨터에는 웹 서버와 관련된 그 어떤 찌꺼기도 남지 않습니다. 이것이 도커의 핵심입니다. 한 줄이면 설치되고 한 줄이면 흔적 없이 사라집니다.

## 5. 자주 쓰는 명령어

도커를 능숙하게 다루기 위해 자주 사용하게 될 명령어들입니다.

### 실행 중인 목록 보기
```bash
docker ps
```
현재 내 컴퓨터에서 돌아가고 있는 컨테이너 상자들의 목록을 보여줍니다. 이름, 상태, 어떤 포트를 쓰고 있는지 확인할 수 있습니다.

### 모든 목록 보기
```bash
docker ps -a
```
실행 중인 상자뿐만 아니라 멈춰 있는 상자까지 모두 보여줍니다. 이전에 실행했다가 종료된 기록들을 확인할 수 있습니다.

### 저장된 레시피 목록 보기
```bash
docker images
```
내 컴퓨터에 내려받아 놓은 이미지(레시피)들의 목록을 보여줍니다. 각각의 레시피가 차지하는 용량도 확인할 수 있습니다.

### 기록 확인하기
```bash
docker logs my-web
```
상자 안에서 어떤 일이 일어나고 있는지 출력되는 메시지들을 확인합니다. 프로그램이 제대로 작동하지 않을 때 원인을 파악하는 가장 중요한 도구입니다.

### 상자 안으로 들어가기
```bash
docker exec -it my-web bash
```
실행 중인 컨테이너 상자 안으로 직접 들어가서 명령어를 입력할 수 있게 해줍니다. 상자 안의 파일 시스템을 직접 살펴보고 싶을 때 사용합니다.

## 6. 깨끗하게 정리하기

실습을 하다 보면 내 컴퓨터에 수많은 상자와 레시피들이 쌓이게 됩니다. 공간을 확보하기 위해 주기적으로 청소해 주는 것이 좋습니다.

- **모든 상자 멈추기**: `docker stop $(docker ps -q)` 명령어로 현재 돌아가는 모든 컨테이너를 한꺼번에 중지시킵니다.
- **모든 상자 버리기**: `docker rm $(docker ps -aq)` 명령어로 멈춰 있는 모든 컨테이너를 삭제합니다.
- **모든 레시피 지우기**: `docker rmi $(docker images -q)` 명령어로 다운로드한 이미지들을 전부 삭제합니다.
- **한꺼번에 대청소하기**: `docker system prune -a` 명령어를 입력하면 사용하지 않는 모든 컨테이너, 이미지, 네트워크 설정을 한 번에 깨끗이 지워줍니다. 아주 강력한 명령어이므로 주의해서 사용해야 합니다.

## 7. 자주 만나는 문제

도커를 사용하다 보면 마주칠 수 있는 몇 가지 상황과 해결 방법입니다.

1. **명령어를 찾을 수 없다는 오류**: `docker: command not found`라는 메시지가 나온다면 도커 데스크탑 프로그램 자체가 실행 중인지 확인해야 합니다. 고래 아이콘이 트레이에 떠 있는지 살펴보세요.
2. **포트가 이미 사용 중이라는 오류**: `port is already allocated`는 내가 사용하려는 포트 번호를 다른 프로그램이나 이미 실행 중인 다른 도커 컨테이너가 점유하고 있을 때 발생합니다. 실행 중인 컨테이너를 중지하거나 포트 번호를 바꾸어 실행해야 합니다.
3. **권한 거부 오류**: 리눅스 환경에서 도커를 사용할 때 자주 발생합니다. 명령어 앞에 관리자 권한을 부여하는 수식어를 붙이거나 사용자를 도커 그룹에 등록해야 합니다. 윈도우나 맥에서는 드물게 나타나지만 발생 시 관리자 권한으로 터미널을 다시 열어보시기 바랍니다.
4. **WSL 관련 오류**: 윈도우에서 도커를 처음 실행할 때 리눅스 커널 업데이트가 필요하다는 메시지가 뜰 수 있습니다. 화면에 표시된 주소로 접속해 업데이트 패키지를 설치하면 바로 해결할 수 있습니다.
5. **컨테이너가 실행되자마자 죽는 경우**: `docker ps`를 쳤을 때 목록에 나타나지 않는다면 프로그램 실행 도중 오류가 발생한 것입니다. `docker logs` 명령어를 통해 어떤 오류 메시지가 남았는지 확인해 보아야 합니다. 설정값이 잘못되었거나 필수 파일이 누락된 경우가 대부분입니다.
6. **저장 공간 부족**: 이미지를 계속 내려받다 보면 나중에 디스크 용량이 부족해집니다. 앞서 설명한 대청소 명령어를 통해 주기적으로 관리해 주어야 합니다.

## 8. 삭제 및 되돌리기

도커 사용을 완전히 중단하거나 처음 상태로 되돌리고 싶을 때의 절차입니다.

### 도커 데스크탑 완전 삭제

윈도우에서는 제어판의 프로그램 추가/제거에서 도커 데스크탑을 선택해 제거합니다. 맥에서는 응용 프로그램 폴더에서 고래 아이콘을 휴지통으로 버립니다. 하지만 이렇게만 해서는 내부적으로 생성된 이미지나 컨테이너 데이터가 완전히 지워지지 않을 수 있습니다. 삭제 전에 반드시 `docker system prune -a --volumes` 명령어를 실행하여 데이터를 먼저 비우는 것을 권장합니다.

### 환경만 되돌리기

프로그램은 유지하되 작업하던 내용만 초기화하고 싶다면 이미지와 컨테이너만 정리하면 됩니다. 6번 항목에서 설명한 명령어들을 순서대로 실행하면 마치 방금 도커를 설치한 것과 같은 깨끗한 상태로 돌아갈 수 있습니다.

## 9. 정리

도커는 환경 설정의 고통에서 우리를 해방해 주는 혁신적인 도구입니다. 이제 우리는 내 컴퓨터의 설정이 꼬일까 걱정하지 않고 마음껏 새로운 기술을 시도해 볼 수 있습니다.

### 핵심 명령어 요약

| 용도 | 명령어 |
| :--- | :--- |
| 서비스 실행 | `docker run [옵션] 이미지이름` |
| 목록 확인 | `docker ps` |
| 서비스 중지 | `docker stop 이름` |
| 서비스 삭제 | `docker rm 이름` |
| 레시피 삭제 | `docker rmi 이미지이름` |
| 전체 초기화 | `docker system prune -a` |

### 작업 흐름 한눈에 보기

1. **이미지 검색 및 확보**: 레지스트리에서 필요한 레시피를 찾습니다.
2. **컨테이너 생성 및 실행**: 이미지를 바탕으로 독립된 상자를 만들어 실행합니다.
3. **사용 및 실습**: 웹 서버나 데이터베이스를 자유롭게 이용합니다.
4. **중지 및 삭제**: 작업이 끝나면 상자를 버려 내 컴퓨터를 깨끗하게 유지합니다.

이번 스터디에서 도커를 환경 되돌리기 레이어로 사용하는 이유는 명확합니다. 실습 도중 어떤 실수를 하더라도 상자만 버리고 다시 만들면 1초 만에 정상적인 상태로 복구할 수 있기 때문입니다. 이제 도커와 함께 두려움 없이 코딩의 세계를 탐험해 보시기 바랍니다.
