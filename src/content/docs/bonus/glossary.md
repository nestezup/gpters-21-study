# 21기 스터디 용어정리

> 바이브코딩 입문자를 위한 핵심 용어 설명.
> 스터디에서 사용하는 도구와 개념을 한눈에 정리합니다.

---

## 기본 개념

### 터미널 (Terminal)
컴퓨터에 **글자로 명령을 내리는 창**. 마우스로 클릭하는 대신, 텍스트를 입력해서 파일을 만들거나 프로그램을 실행한다.
- Windows: PowerShell, 명령 프롬프트
- macOS: 터미널(Terminal.app)

### 패키지 관리자 (Package Manager)
프로그램을 **한 줄 명령어로 설치·삭제·업데이트**하는 도구. 앱스토어의 터미널 버전이라고 생각하면 된다.
- Windows: scoop
- macOS: Homebrew (brew)

### scoop
**Windows용 패키지 관리자**. 복잡한 설치 과정 없이 터미널에서 `scoop install 앱이름` 한 줄이면 프로그램 설치가 끝난다. 내 컴퓨터 환경을 깔끔하게 유지하면서 도구들을 관리하기 좋다.

### Homebrew (brew)
**macOS용 패키지 관리자**. 맥 사용자들이 가장 많이 쓰는 도구로, `brew install 앱이름` 명령어로 필요한 개발 도구를 순식간에 설치할 수 있다.

### 환경 변수 (Environment Variable)
컴퓨터 전체에서 언제든 꺼내 쓸 수 있도록 저장해둔 **공통 설정값**. 예를 들어 "어떤 폴더에 프로그램이 있는지" 미리 등록해두면, 터미널 어디서든 그 프로그램 이름을 불러서 실행할 수 있다.

### 셸 설정 파일 (.zshrc, $PROFILE)
터미널이 켜질 때마다 **자동으로 읽어오는 설정 메모장**. 터미널의 테마를 바꾸거나, 환경 변수를 영구적으로 저장할 때 사용한다. "터미널의 시작 설정"이라고 이해하면 된다.

### 플레이그라운드 (Playground)
**자유롭게 실험할 수 있는 연습 환경**. 망가뜨려도 되돌릴 수 있으니 부담 없이 시도할 수 있다.

### 바이브코딩 (Vibe Coding)
AI에게 자연어로 원하는 것을 설명하면, AI가 코드를 작성해주는 방식의 개발. 코드를 한 줄씩 외워서 타이핑하는 게 아니라, **"이런 거 만들어줘"**라고 말하면 만들어진다.

## 코드 레이어: Git

### Git
코드의 **변경 이력을 기록하는 도구**. 세이브 포인트를 만들어 두고, 언제든 과거 상태로 돌아갈 수 있다.

### 커밋 (Commit)
Git에서 만드는 **하나의 세이브 포인트**. "여기까지는 잘 되니까 저장해두자"라는 의미.
```
git commit -m "로그인 기능 완성"
```

### 스테이징 (Staging)
커밋(저장)을 하기 전에 **"이 파일들을 저장 목록에 올리겠다"고 골라내는 단계**. `git add` 명령어를 쓰면 파일들이 스테이징 영역(장바구니 같은 곳)에 담긴다.

### 리포지토리 (Repository, 줄여서 repo)
Git이 관리하는 **프로젝트 폴더**. 코드 + 모든 변경 이력이 들어 있다.
- 로컬 리포지토리: 내 컴퓨터에 있는 것
- 원격 리포지토리: GitHub 같은 서버에 있는 것

### .git 폴더
`git init`을 하면 생기는 **비밀 창고 같은 숨김 폴더**. 프로젝트의 모든 과거 기록과 세이브 포인트가 이 안에 압축되어 들어있다. 이 폴더를 지우면 Git 기록도 사라지니 주의해야 한다.

### .gitignore
Git이 **기록하지 않고 무시할 파일 목록**. 보안이 중요한 API 키나, 용량만 차지하고 다시 만들 수 있는 설정 파일들을 여기에 적어두면 Git이 알아서 제외해준다.

### 브랜치 (Branch)
하나의 프로젝트에서 **갈래를 나눠서 작업**하는 것. 원본을 건드리지 않고 실험할 수 있다.

## 환경 레이어: 런타임 & 버전 관리

### Node.js
JavaScript를 브라우저 밖에서도 실행할 수 있게 해주는 **실행 환경(런타임)**. 웹 개발, 서버, AI 도구 등 많은 곳에서 사용한다.

### npm (Node Package Manager)
Node.js를 설치하면 함께 따라오는 **자바스크립트용 부품 가게**. 전 세계 개발자들이 만들어둔 유용한 코드 뭉치(패키지)를 가져와서 내 프로젝트에 바로 쓸 수 있게 해준다.

### fnm (Fast Node Manager)
Node.js의 **버전을 여러 개 설치하고 전환**하는 도구. 프로젝트마다 다른 Node.js 버전이 필요할 때 유용하다.
```
fnm install 22
fnm use 22
```

### LTS (Long Term Support)
제작사에서 **오랫동안 책임지고 관리해주는 안정적인 버전**. 최신 기능보다는 "절대 고장 나지 않는 안정성"이 중요할 때 사용하며, 입문자에게 가장 권장되는 버전이다.

### 런타임 (Runtime)
프로그래밍 언어로 작성한 코드를 **실제로 실행해주는 환경**. Node.js는 JavaScript의 런타임이다.

## 환경 레이어: 컨테이너

### Docker
앱과 그 앱이 필요로 하는 환경을 **통째로 묶어서 실행**하는 도구. 내 컴퓨터 환경을 더럽히지 않고 프로그램을 실행할 수 있다.

### Docker Desktop
Docker를 마우스 클릭만으로 **쉽게 관리할 수 있게 해주는 앱**. 터미널 명령어가 익숙하지 않아도 컨테이너가 잘 돌아가는지 눈으로 확인하고 제어할 수 있다.

### 컨테이너 (Container)
Docker가 만드는 **격리된 실행 환경**. 하나의 컴퓨터 안에 독립된 작은 방을 여러 개 만든다고 생각하면 된다. 필요 없으면 방째로 삭제하면 끝.

### 이미지 (Image)
컨테이너를 만들기 위한 **설계도(레시피)**. 이미지 하나로 똑같은 컨테이너를 몇 개든 만들 수 있다.
```
docker run nginx        # nginx 이미지로 컨테이너 생성+실행
docker rm -f my-nginx   # 컨테이너 삭제
```

### Docker Hub / 레지스트리 (Registry)
Docker 이미지들이 모여 있는 **온라인 앱스토어**. 남들이 미리 만들어둔 이미지를 검색해서 다운로드(pull)받거나, 내가 만든 이미지를 올려서 공유할 수 있다.

### 포트 (Port)
컴퓨터로 들어오는 **데이터의 출입문 번호**. 한 컴퓨터에서 여러 프로그램이 네트워크를 쓸 때, "8080번 문은 이 앱이 쓴다"라고 정해서 서로 겹치지 않게 구분해준다.

## OS 레이어: 가상화

### 가상화 (Virtualization)
하나의 컴퓨터 안에 **또 다른 컴퓨터를 소프트웨어로 만드는 기술**. 진짜 컴퓨터처럼 OS를 설치하고 사용할 수 있다.

### 가상머신 (Virtual Machine, VM)
가상화로 만들어진 **가상의 컴퓨터**. 실제 하드웨어 없이 OS를 통째로 실행한다.

### 스냅샷 (Snapshot)
가상머신의 **현재 상태를 통째로 저장**한 것. 사진 찍듯이 그 순간을 보존하고, 나중에 그 시점으로 정확히 되돌아갈 수 있다.

### Hyper-V
Windows에 기본으로 들어있는 **가상화 엔진**. 별도의 프로그램을 설치하지 않아도 Windows 설정만으로 가상 컴퓨터를 만들고 실행할 수 있게 해주는 기술이다.

### ISO 파일
운영체제(OS)를 설치하기 위한 **가상 설치 디스크**. 예전의 윈도우 설치 CD를 파일 하나로 만들어둔 것이라고 생각하면 된다. 가상머신에 OS를 깔 때 이 파일이 필요하다.

### UTM
macOS에서 사용하는 **무료 가상화 앱**. Mac 위에서 Windows, Linux 등을 실행할 수 있다.

### VMware
Windows/Mac/Linux에서 사용하는 **가상화 소프트웨어**. 무료 버전(VMware Workstation Player)으로 충분히 활용 가능.

### WSL (Windows Subsystem for Linux)
Windows 안에서 **Linux를 바로 실행**하는 기능. 가상머신보다 가볍고 빠르다. Windows 사용자가 Linux 명령어를 쓸 수 있게 해준다.
```
wsl --install          # 설치
wsl --export Ubuntu backup.tar   # 백업
wsl --unregister Ubuntu          # 삭제 (되돌리기)
wsl --import Ubuntu C:\wsl backup.tar  # 복원
```

## AI 도구

### Claude Code
터미널에서 동작하는 **AI 코딩 어시스턴트**. 자연어로 명령하면 파일을 만들고, 코드를 수정하고, 터미널 명령어를 실행해준다.

### Cursor
**AI가 내장된 코드 에디터**. VS Code를 기반으로 만들어졌고, 코드를 보면서 AI에게 수정을 요청할 수 있다.

## 레이어별 "되돌리기" 요약

| 레이어 | 도구 | 되돌리기 방법 |
|--------|------|--------------|
| 코드 | Git | `git checkout`, `git reset` → 커밋 단위로 복원 |
| 환경 | Docker | 컨테이너 삭제 후 이미지로 재생성 |
| 환경 | WSL | `wsl --unregister` 후 재설치 또는 백업 복원 |
| 환경 | fnm | `fnm uninstall` 후 다른 버전 설치 |
| OS | UTM/VMware/Hyper-V | 스냅샷으로 원하는 시점으로 복원 |
