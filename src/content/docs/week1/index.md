# Git: 코드의 세이브 포인트 만들기

## 1. Git이 뭔가요?

우리가 컴퓨터로 중요한 문서를 작성하거나 게임을 할 때를 떠올려 보겠습니다. 어려운 적과 싸우기 직전이나 아주 긴 보고서를 반쯤 썼을 때 우리는 본능적으로 저장 버튼을 누릅니다. 만약 저장하지 않은 상태에서 컴퓨터가 꺼지거나 실수로 내용을 다 지워버린다면 그동안의 노력이 모두 물거품이 되기 때문입니다.

코딩도 마찬가지입니다. 코드를 한 줄씩 적어 내려가다 보면 어제 짠 코드가 더 나았다는 생각이 들기도 하고, 새로 추가한 기능 때문에 프로그램이 아예 작동하지 않는 상황을 마주하기도 합니다. 이때 우리는 과거의 어느 시점으로 돌아가고 싶어집니다.

Git이 없던 시절에는 파일 이름을 바꿔가며 버전을 관리했습니다. 보고서_최종.docx, 보고서_최종_진짜최종.docx, 보고서_최종_진짜진짜최종_수정완료.docx 같은 식으로 말입니다. 하지만 파일이 수백 개로 늘어나면 어떤 파일이 최신인지, 무엇이 바뀌었는지 확인하기가 무척 어렵습니다.

Git은 바로 이 문제를 해결해 주는 도구입니다. 여러분의 프로젝트 폴더 전체를 하나의 게임 세계라고 생각한다면, Git은 여러분이 원하는 순간마다 세이브 포인트를 만들 수 있게 도와줍니다. 언제든 과거의 세이브 포인트로 돌아가서 다시 시작할 수 있고, 누가 언제 어떤 내용을 수정했는지도 한눈에 보여줍니다.

## 2. Git 설치하기

컴퓨터에 Git을 설치하는 방법은 운영체제마다 다릅니다. 각자의 환경에 맞는 명령어를 입력해 보겠습니다.

### Windows (scoop 이용)
이미 scoop이라는 도구를 사용하고 있다면 아래 명령어를 입력하여 설치할 수 있습니다.
```powershell
scoop install git
```
만약 scoop이 설치되어 있지 않다면 별도의 패키지 관리자 가이드를 참고하여 먼저 설치해 주시기 바랍니다.

### Windows (winget 이용)
Windows 10이나 11을 사용 중이라면 기본으로 포함된 winget을 사용할 수 있습니다.
```powershell
winget install --id Git.Git -e --source winget
```

### macOS (Homebrew 이용)
맥 사용자는 Homebrew를 통해 설치하는 것이 편리합니다.
```bash
brew install git
```

### 설치 확인
설치가 완료되었다면 터미널 창을 열고 아래 명령어를 입력해서 숫자가 나오는지 확인합니다.
```bash
git --version
```
숫자가 나타난다면 이제 Git을 사용할 준비가 끝났습니다.

## 3. Git 초기 설정

세이브 포인트를 만들 때 가장 중요한 것은 누가 이 세이브를 했는지 기록하는 일입니다. 여러 사람이 함께 작업할 때는 특히 더 그렇습니다. Git에게 여러분의 이름과 이메일 주소를 알려주어야 합니다.

```bash
git config --global user.name "내이름"
git config --global user.email "이메일@example.com"
```
여기서 입력한 정보는 세이브 기록에 영구히 남게 됩니다. 설정을 마친 뒤에는 아래 명령어로 정보가 잘 들어갔는지 확인해 봅니다.
```bash
git config --list
```

## 4. 처음 사용해보기: 첫 번째 세이브 포인트 만들기

이제 실제로 코드를 작성하고 세이브 포인트를 만드는 과정을 연습해 보겠습니다. 하나하나 차근차근 따라와 주세요.

### 1) 연습용 폴더 만들기
작업할 공간을 먼저 마련합니다.
```bash
mkdir my-first-project
cd my-first-project
```

### 2) Git 시작하기
이제 이 폴더를 Git이 지켜보게 만들어야 합니다.
```bash
git init
```
이 명령어를 입력하면 폴더 안에 .git이라는 숨겨진 폴더가 생깁니다. 이 폴더는 여러분이 만드는 모든 세이브 포인트와 변경 기록을 담아두는 비밀 창고입니다. 이제부터 이 폴더 안에서 일어나는 모든 변화를 Git이 기록할 수 있습니다.

### 3) 파일 만들고 저장하기
내용이 들어있는 파일을 하나 만들어 보겠습니다.
```bash
echo "안녕하세요" > hello.txt
```

### 4) 변경 사항 확인하기
방금 파일을 만들었습니다. Git은 이 상황을 어떻게 보고 있을까요?
```bash
git status
```
명령어를 입력하면 빨간색 글씨로 파일 이름이 보일 것입니다. 이것은 Git이 이 파일을 발견했지만, 아직 세이브 포인트에 넣을 준비가 되지 않았다는 뜻입니다.

### 5) 세이브 준비 (스테이징)
세이브 포인트를 만들기 전에는 어떤 파일을 세이브에 포함할지 골라야 합니다. 이 과정을 스테이징이라고 부릅니다.
```bash
git add hello.txt
```
다시 `git status`를 입력해 보세요. 파일 이름이 초록색으로 변했을 것입니다. 이제 세이브할 준비가 완료되었습니다.

### 6) 세이브 (커밋)
이제 드디어 세이브 포인트를 만듭니다. Git에서는 이것을 커밋이라고 부릅니다.
```bash
git commit -m "첫 번째 파일 추가"
```
큰따옴표 안에는 이 세이브 포인트가 어떤 내용을 담고 있는지 설명을 적습니다. 나중에 기록을 보고 어떤 작업이었는지 알 수 있도록 정성껏 적는 습관을 들이는 것이 좋습니다.

### 7) 세이브 기록 확인
우리가 만든 세이브 포인트가 잘 저장되었는지 확인해 봅니다.
```bash
git log --oneline
```
여러분이 방금 적은 메시지와 함께 고유한 번호가 나타난다면 성공입니다.

## 5. 되돌리기 실습

Git을 사용하는 가장 큰 이유는 잘못했을 때 되돌릴 수 있기 때문입니다. 실제로 파일을 망가뜨려 보고 다시 고쳐보겠습니다.

### 1) 파일 수정하기
원래 있던 내용을 지우고 엉뚱한 내용을 넣어 봅니다.
```bash
echo "파일을 망가뜨렸어요" > hello.txt
```
파일을 열어보면 원래 있던 안녕하세요라는 말은 사라지고 없습니다.

### 2) 수정 전으로 되돌리기
아직 세이브 포인트(커밋)를 만들지 않았다면 마지막 세이브 상태로 바로 되돌릴 수 있습니다.
```bash
git checkout -- hello.txt
```
다시 파일을 확인해 보세요. 마법처럼 원래의 안녕하세요라는 내용이 돌아와 있을 것입니다.

### 3) 커밋한 상태로 되돌리기
만약 실수로 잘못된 내용을 세이브 포인트로 만들어버렸다면 어떡해야 할까요? 이때는 `git reset` 명령어를 사용합니다. 이 명령어는 시간을 과거로 돌려버리는 것과 같습니다. 특정 시점의 세이브 번호를 지정하면 그 이후의 기록을 지우고 과거로 돌아갈 수 있습니다. 이 과정은 매우 강력하므로 신중하게 사용해야 합니다.

## 6. 자주 만나는 문제

Git을 처음 쓰다 보면 당황스러운 상황을 자주 겪게 됩니다. 대표적인 사례 다섯 가지를 미리 알아두면 도움이 됩니다.

### fatal: not a git repository
이 오류는 `git init`을 하지 않은 폴더에서 Git 명령어를 쓰려고 할 때 발생합니다. 현재 내가 작업 폴더 안에 제대로 들어와 있는지, 초기화 명령어를 입력했는지 확인해 보세요.

### 커밋 메시지 입력 화면에 갇혔을 때
`git commit`을 할 때 뒤에 `-m` 옵션을 빼먹으면 검은 화면에 이상한 글자들이 가득한 화면이 나타납니다. 이것은 vim이라는 편집기입니다. 당황하지 말고 키보드의 Esc를 누른 뒤 `:q!`를 입력하고 Enter를 누르면 빠져나올 수 있습니다. 메시지를 함께 적으려면 반드시 `-m "내용"` 형식을 지켜주세요.

### nothing to commit
세이브할 내용이 없다는 뜻입니다. 파일을 수정하고 나서 `git add`를 하지 않았거나, 마지막 세이브 이후로 바뀐 내용이 전혀 없을 때 이런 메시지가 나옵니다.

### 한글 파일명 깨짐 문제
Windows 환경에서 한글 이름을 가진 파일을 만들면 Git에서 글자가 깨져 보일 수 있습니다. 터미널의 설정을 변경하거나 Git의 설정을 조정해서 해결할 수 있는 문제입니다. 가급적 파일 이름은 영어로 짓는 것이 혼란을 줄이는 방법이 되기도 합니다.

### .gitignore가 뭔가요?
프로젝트를 하다 보면 기록에 남기고 싶지 않은 파일들이 생깁니다. 비밀번호가 담긴 파일이나 용량이 너무 큰 설정 파일 같은 것들입니다. 이런 파일의 이름을 `.gitignore`라는 파일 안에 적어두면 Git이 그 파일들을 무시하고 지나갑니다.

## 7. 삭제와 되돌리기

작업을 하다가 프로젝트를 처음부터 다시 시작하고 싶을 수도 있습니다. 이때 폴더 안에 있는 `.git` 폴더를 삭제하면 그동안 쌓아온 모든 세이브 포인트와 기록이 한순간에 사라집니다.

이것은 프로젝트 폴더 자체를 삭제하는 것과는 다릅니다. 프로젝트 폴더를 지우면 파일 자체가 없어지지만, `.git` 폴더만 지우면 현재 파일들은 그대로 남되 Git이 관리하던 과거의 기록들만 사라집니다. 즉, 세이브 파일만 지우고 게임은 계속 켜두는 것과 비슷합니다. 기록을 지우는 일은 되돌릴 수 없으니 주의해야 합니다.

## 8. 정리

Git의 흐름은 명확합니다. 파일을 만들고, 세이브할 준비를 하고, 설명을 붙여서 세이브 포인트로 저장하는 과정의 반복입니다.

### 핵심 명령어 모음

| 명령어 | 역할 | 비유 |
| :--- | :--- | :--- |
| git init | Git 저장소 시작 | 게임 시작 및 자동저장 기능 활성화 |
| git status | 상태 확인 | 현재 세이브되지 않은 변경사항 확인 |
| git add [파일] | 스테이징 | 세이브할 항목 바구니에 담기 |
| git commit -m "[메시지]" | 커밋 | 설명 적고 세이브 포인트 생성 |
| git log | 기록 확인 | 지금까지의 세이브 목록 열람 |
| git checkout -- [파일] | 파일 되돌리기 | 마지막 세이브 시점으로 파일 복구 |

처음에는 이 과정이 낯설고 번거롭게 느껴질 수 있습니다. 하지만 세이브 포인트 없이 거대한 성을 쌓아 올리는 것보다, 중간중간 깃발을 꽂으며 나아가는 것이 훨씬 마음 편하고 안전한 길입니다. 이제 여러분의 첫 번째 프로젝트에 튼튼한 세이브 포인트를 만들어 보세요.
