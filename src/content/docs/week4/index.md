# UTM, VMware, Hyper-V: 운영체제 통째로 되돌리기

우리는 가끔 시간을 되돌리고 싶을 때가 있습니다. 중요한 파일을 실수로 지웠거나 시스템 설정을 잘못 건드려 컴퓨터가 엉망이 되었을 때 말입니다. 코딩을 시작하는 분들에게 컴퓨터 환경 설정은 가장 큰 벽 중 하나입니다. 설정을 하나만 잘못해도 컴퓨터 전체가 작동하지 않을까 봐 걱정하는 분들이 많습니다. 이런 걱정을 덜어주기 위해 등장한 기술이 바로 가상화입니다. 이번 글에서는 가상화가 무엇인지 알아보고 대표적인 도구인 UTM과 VMware를 사용하는 방법을 살펴보겠습니다.

## 1. 가상화가 뭔가요?

가상화는 하나의 물리적인 컴퓨터 안에 또 다른 독립된 컴퓨터를 만드는 기술입니다. 우리가 흔히 사용하는 노트북이나 데스크톱 안에 소프트웨어로 구현된 가상의 컴퓨터를 한 대 더 설치한다고 이해하면 됩니다. 마치 투명한 연습장을 하나 준비해서 그 위에 마음껏 그림을 그리고 틀리면 페이지를 찢어버리거나 이전 상태로 되돌리는 것과 비슷합니다.

진짜 컴퓨터를 새로 한 대 사는 것과 가상머신을 만드는 것 사이에는 명확한 차이가 있습니다. 실제 컴퓨터를 사려면 공간이 필요하고 비용도 많이 듭니다. 하지만 가상머신은 현재 사용 중인 컴퓨터의 자원을 나누어 사용합니다. 내 컴퓨터의 두뇌인 CPU와 기억 공간인 RAM 그리고 저장소의 일부를 빌려와서 새로운 환경을 구성하는 방식입니다.

이 기술은 코딩 연습장으로 활용하기에 매우 훌륭합니다. 운영체제를 통째로 망가뜨려도 실제 내 컴퓨터에는 아무런 지장이 없습니다. 가상머신 안에서 바이러스 실험을 하거나 시스템 폴더를 삭제해도 가상머신만 종료하면 실제 환경은 안전합니다.

많은 분이 Docker와 가상머신을 혼동하기도 합니다. Docker는 주로 애플리케이션 하나가 돌아갈 수 있는 최소한의 환경을 격리하는 방식입니다. 반면 가상머신은 윈도우나 리눅스 같은 운영체제 자체를 통째로 설치하여 독립된 하드웨어처럼 작동하게 합니다. 따라서 시스템의 깊은 곳까지 공부하고 실습하고 싶다면 가상머신이 더 적합한 선택입니다.

## 2. 어떤 도구를 쓸까?

어떤 도구를 선택할지는 현재 여러분이 사용하고 있는 컴퓨터의 운영체제에 따라 달라집니다.

첫 번째 후보인 UTM은 애플 실리콘을 탑재한 맥 사용자에게 최적화된 도구입니다. 맥 전용으로 제작되었으며 오픈 소스로 공개되어 있어 무료로 사용할 수 있습니다. 맥의 하드웨어 성능을 효율적으로 끌어다 쓰기 때문에 부드러운 환경을 제공합니다.

두 번째 후보인 VMware는 세계적으로 널리 쓰이는 가상화 도구입니다. 윈도우와 맥 그리고 리눅스까지 다양한 환경을 지원합니다. 윈도우 사용자라면 무료로 제공되는 VMware Workstation Player를 선택하는 것이 보편적입니다. 맥 사용자를 위한 VMware Fusion도 존재하지만 최근 정책 변화가 잦으므로 설치 시점에 라이선스를 확인해야 합니다.

세 번째 후보인 Hyper-V는 윈도우에 기본 내장된 가상화 기능입니다. 별도의 프로그램을 설치할 필요 없이 윈도우의 기능을 켜기만 하면 바로 사용할 수 있습니다. 마이크로소프트가 직접 만든 만큼 윈도우와의 궁합이 매우 좋고, 성능 면에서도 다른 도구들보다 유리한 점이 있습니다. 다만 윈도우 10/11 Pro, Enterprise, Education 에디션에서만 사용할 수 있고 Home 에디션에서는 지원되지 않습니다.

정리하면 맥을 사용하고 있다면 UTM을, 윈도우 Pro 이상을 사용하고 있다면 Hyper-V를, 윈도우 Home을 사용하고 있다면 VMware를 권장합니다. 이 외에도 무료 도구인 VirtualBox가 있지만 위 세 가지가 각각의 환경에서 더 안정적인 경험을 제공합니다.

## 3. UTM 설치하기 (macOS)

맥 사용자라면 두 가지 방법 중 하나를 선택해 UTM을 설치할 수 있습니다.

### Homebrew로 설치하기
터미널을 열고 아래 명령어를 입력합니다.

```bash
brew install --cask utm
```

명령어를 입력하고 잠시 기다리면 자동으로 설치가 완료됩니다.

### 직접 다운로드하여 설치하기
웹 브라우저에서 공식 홈페이지(https://mac.getutm.app/)에 접속합니다. 화면에 보이는 다운로드 버튼을 눌러 설치 파일을 내려받습니다. 내려받은 .dmg 파일을 열고 UTM 아이콘을 Applications 폴더로 드래그하면 설치가 끝납니다.

### 필수 조건
UTM을 실행하려면 macOS 11 이상 버전이 필요합니다. 최신 맥뿐만 아니라 인텔 프로세서를 탑재한 구형 맥에서도 작동하므로 설치 전에 시스템 정보를 확인하시기 바랍니다.

## 4. VMware 설치하기 (Windows)

윈도우 사용자는 VMware Workstation Player를 설치해야 합니다. 공식 웹사이트의 다운로드 페이지에서 운영체제에 맞는 설치 파일을 내려받습니다.

설치 과정에서 유의할 점이 몇 가지 있습니다. 설치 파일을 실행하고 안내에 따라 다음 버튼을 누르다 보면 비상업적 용도로 사용할 것인지 묻는 창이 나옵니다. 개인적인 학습 목적이라면 이 옵션을 선택하여 무료로 이용할 수 있습니다. 설치가 끝나면 컴퓨터를 재시작해야 할 수도 있으니 작업 중인 파일이 있다면 미리 저장해 두어야 합니다.

하드웨어 권장 사양으로는 최소 8GB 이상의 메모리가 필요합니다. 가상머신 자체가 메모리를 많이 차지하기 때문에 여유 공간이 넉넉할수록 쾌적하게 실습할 수 있습니다. 또한 CPU 설정에서 가상화 기능이 켜져 있어야 하므로 미리 확인이 필요합니다.

## 5. Hyper-V 설치하기 (Windows Pro 이상)

Hyper-V는 윈도우에 이미 들어 있는 기능이므로 별도의 설치 파일을 내려받을 필요가 없습니다. 숨겨져 있는 기능을 켜주기만 하면 됩니다.

### 내 윈도우 에디션 확인하기

Hyper-V를 사용하기 전에 내 윈도우가 Pro 에디션 이상인지 먼저 확인해야 합니다. 키보드에서 윈도우 키와 I 키를 동시에 누르면 설정 화면이 열립니다. 시스템 메뉴에서 정보를 선택하면 윈도우 사양이 보입니다. 여기서 에디션이 Pro, Enterprise, Education 중 하나라면 Hyper-V를 사용할 수 있습니다. Home이라고 적혀 있다면 아쉽지만 Hyper-V 대신 VMware를 사용하시기 바랍니다.

### 기능 켜기

파워쉘을 관리자 권한으로 실행한 뒤 아래 명령어를 입력합니다.

```powershell
Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V -All
```

명령어를 실행하면 필요한 구성 요소를 설치한 뒤 재부팅을 요청합니다. 재부팅이 완료되면 시작 메뉴에서 Hyper-V Manager를 검색하여 실행할 수 있습니다.

파워쉘 대신 그래픽 화면에서 설정하고 싶다면 제어판에서 프로그램 및 기능을 선택하고, 왼쪽의 Windows 기능 켜기/끄기를 클릭합니다. 목록에서 Hyper-V 항목의 체크박스를 선택한 뒤 확인을 누르면 동일하게 설치가 진행됩니다.

### Hyper-V에서 가상머신 만들기

1. Hyper-V Manager를 실행합니다.
2. 오른쪽 메뉴에서 새로 만들기 항목의 가상 컴퓨터를 선택합니다.
3. 이름을 정하고 저장 위치를 확인합니다.
4. 세대 선택에서는 2세대를 선택하는 것이 권장됩니다. 최신 리눅스 배포판은 2세대를 잘 지원합니다.
5. 메모리는 4096MB(4GB) 이상을 할당합니다.
6. 네트워크 어댑터에서 Default Switch를 선택하면 인터넷이 자동으로 연결됩니다.
7. 가상 하드 디스크 크기를 지정합니다. 실습용이라면 30GB에서 40GB 정도면 충분합니다.
8. 설치 옵션에서 미리 내려받은 Ubuntu ISO 파일을 연결합니다.
9. 마침을 누른 뒤 목록에서 해당 가상머신을 선택하고 시작 버튼을 클릭하면 설치가 시작됩니다.

### Hyper-V의 체크포인트

Hyper-V에서는 스냅샷을 체크포인트라고 부릅니다. 이름만 다를 뿐 동일한 기능입니다. 가상머신 목록에서 마우스 오른쪽 버튼을 누르면 체크포인트 만들기 메뉴가 나타납니다. 되돌리고 싶을 때는 체크포인트 목록에서 원하는 시점을 선택하고 적용을 누르면 됩니다.

### Hyper-V를 사용할 때 알아두어야 할 점

Hyper-V를 켜면 윈도우 자체가 하이퍼바이저 위에서 동작하게 됩니다. 이 때문에 VMware나 VirtualBox 같은 다른 가상화 도구와 동시에 사용할 때 충돌이 발생할 수 있습니다. 만약 다른 가상화 도구를 함께 쓸 계획이라면 이 점을 미리 고려하시기 바랍니다. Hyper-V 기능을 끄려면 같은 방법으로 체크박스를 해제하고 재부팅하면 됩니다.

## 6. 가상머신 만들어보기

이제 가상 컴퓨터 안에 운영체제를 설치해 보겠습니다. 가장 많이 쓰이는 리눅스인 Ubuntu를 기준으로 설명하겠습니다.

### Ubuntu 설치 준비
먼저 Ubuntu 공식 홈페이지에서 이미지 파일을 내려받아야 합니다. 이 파일은 실제 컴퓨터의 설치 CD와 같은 역할을 합니다. 내 컴퓨터의 CPU 방식에 맞춰 적절한 파일을 선택하시기 바랍니다.

### UTM에서 가상머신 만들기
1. UTM을 실행하고 새 가상머신 만들기 버튼을 클릭합니다.
2. 가상화 방식 선택 창이 나오면 Virtualize를 선택합니다. 이 방식은 내 컴퓨터의 성능을 최대한 활용하게 해줍니다.
3. 리눅스를 선택하고 아까 내려받은 이미지 파일을 연결합니다.
4. CPU 코어 개수와 메모리 크기를 정합니다. 일반적으로 메모리는 4GB 정도 할당하는 것이 좋습니다.
5. 저장 공간의 크기를 정합니다. 실습용이라면 30GB에서 40GB 정도면 충분합니다.
6. 모든 설정을 마치면 재생 버튼을 눌러 설치를 시작합니다. 화면의 안내에 따라 언어와 지역을 선택하면 설치가 진행됩니다.

### VMware에서 가상머신 만들기
과정은 UTM과 매우 흡사합니다. 새로 만들기 버튼을 누르고 이미지 파일을 선택한 뒤 이름과 저장 위치를 지정합니다. 하드웨어 설정 메뉴에서 메모리와 프로세서 개수를 조정할 수 있습니다. 설정이 끝나면 가상머신을 켜서 설치 과정을 마무리합니다.

## 7. 스냅샷: 가상화의 꽃

가상화를 사용하는 가장 핵심적인 이유가 바로 이 스냅샷 기능입니다. 스냅샷은 현재 가상머신의 모든 상태를 사진을 찍듯이 그대로 저장하는 기능입니다. 

### 스냅샷이란 무엇인가요?
게임을 하다가 보스 몬스터와 싸우기 직전에 저장하는 세이브 포인트라고 생각하면 이해가 빠릅니다. 만약 보스에게 패배하더라도 저장한 시점부터 다시 시작할 수 있듯이 가상머신도 스냅샷을 찍어두면 언제든 그 시점으로 돌아갈 수 있습니다.

### 스냅샷 만들기
UTM에서는 가상머신 정보 화면에서 스냅샷 탭을 선택하여 현재 상태를 저장할 수 있습니다. VMware에서는 상단 메뉴에서 스냅샷 찍기 기능을 찾아 이름을 붙여 저장하면 됩니다. "실습 시작 전" 혹은 "패키지 설치 완료"와 같이 구체적인 이름을 붙여두면 나중에 알아보기 편리합니다.

### 일부러 망가뜨려 보기
이제 용기를 내어 가상머신 안에서 아무거나 해보시기 바랍니다. 시스템에 꼭 필요한 파일을 삭제해 보거나 설정값을 터무니없게 바꿔 보십시오. 화면이 나오지 않거나 부팅이 되지 않아도 괜찮습니다. 우리에게는 스냅샷이 있기 때문입니다.

### 되돌리기
문제가 생겼다면 해당 가상머신의 스냅샷 목록에서 아까 저장했던 지점을 선택하고 복원 버튼을 누릅니다. 몇 초만 기다리면 마치 아무 일도 없었다는 듯이 깨끗했던 초기 상태로 돌아옵니다. 이것이 바로 가상화가 제공하는 강력한 안전망입니다. 여러분은 이제 실패를 두려워하지 않고 마음껏 코딩 실습을 할 수 있습니다.

## 8. 자주 만나는 문제

실습 중에 다음과 같은 상황이 발생할 수 있습니다. 당황하지 말고 대응 방법을 확인해 보세요.

첫 번째로 가상머신이 너무 느리게 작동할 때입니다. 이럴 때는 내 컴퓨터의 자원을 가상머신에 너무 많이 할당했거나 반대로 너무 적게 할당했을 가능성이 큽니다. 전체 메모리의 절반 이상을 가상머신에 주면 실제 컴퓨터가 느려져서 전체적인 성능이 떨어집니다. 적절한 균형을 찾는 것이 중요합니다.

두 번째는 가상화 기능이 비활성화되어 있다는 오류입니다. 이는 컴퓨터 본체의 설정인 BIOS에서 가상화 기술 관련 항목이 꺼져 있기 때문입니다. 컴퓨터를 켤 때 특정 키를 눌러 설정 화면에 진입한 뒤 가상화 기능을 켜주어야 합니다.

세 번째는 화면 해상도가 작게 고정되어 불편한 경우입니다. 가상머신 안에 게스트 확장 도구를 설치하면 화면 크기를 자유롭게 조절할 수 있습니다. UTM과 VMware 모두 메뉴 상단에서 관련 도구 설치 기능을 제공합니다.

네 번째는 실제 컴퓨터와 가상머신 사이에 파일을 주고받고 싶을 때입니다. 공유 폴더 기능을 설정하면 마치 하나의 컴퓨터처럼 폴더를 공유할 수 있습니다. 설정 메뉴에서 공유 경로를 지정하면 가상머신 안에서 해당 폴더를 열 수 있습니다.

다섯 번째는 인터넷 연결이 되지 않는 문제입니다. 네트워크 설정이 NAT 방식으로 되어 있는지 확인해 보십시오. 대부분의 환경에서는 이 설정이 가장 안정적으로 인터넷에 연결되도록 도와줍니다.

## 9. 삭제 및 정리

더 이상 가상머신이 필요하지 않다면 깔끔하게 정리하는 과정도 중요합니다.

가상머신을 삭제하려면 목록에서 마우스 오른쪽 버튼을 누르고 삭제를 선택합니다. 이때 주의할 점은 목록에서만 지우는 것이 아니라 실제 파일까지 모두 삭제해야 저장 공간이 회수된다는 사실입니다. 가상머신은 수십 기가바이트의 용량을 차지하므로 사용하지 않는다면 파일을 완전히 지워주는 것이 좋습니다.

프로그램 자체를 지우고 싶다면 맥은 응용 프로그램 폴더에서 UTM을 휴지통으로 버리면 되고 윈도우는 제어판의 프로그램 추가 및 제거 메뉴를 이용합니다. 도구를 삭제해도 가상머신 파일은 남아있을 수 있으니 내 문서 폴더나 지정했던 저장 경로를 다시 한번 확인해 보시기 바랍니다.

## 10. 정리

지금까지 가상화 기술과 이를 돕는 도구들을 살펴보았습니다.

| 비교 항목 | UTM | VMware | Hyper-V |
| :--- | :--- | :--- | :--- |
| 추천 운영체제 | macOS | Windows (Home 포함) | Windows Pro 이상 |
| 비용 | 무료 | 개인 사용 시 무료 | 무료 (윈도우 내장) |
| 설치 방식 | 앱 설치 | 앱 설치 | 윈도우 기능 켜기 |
| 주요 강점 | 애플 실리콘 지원 최적화 | 높은 호환성과 안정성 | 윈도우 네이티브, 높은 성능 |
| 되돌리기 기능 | 스냅샷 | 스냅샷 | 체크포인트 |
| 주의 사항 | 맥 전용 | 특별한 제약 없음 | 다른 가상화 도구와 충돌 가능 |

가상머신의 스냅샷(또는 체크포인트) 기능은 여러분이 새로운 기술을 배울 때 가장 든든한 보호 장비가 되어줄 것입니다. 명령어 한 줄을 잘못 입력해서 시스템이 멈추더라도 우리는 단 몇 초 만에 과거로 돌아갈 수 있습니다. 이 기술을 충분히 활용하여 두려움 없이 코딩의 세계로 뛰어드셨으면 좋겠습니다. 여러분의 컴퓨터는 가상화라는 안전망 안에서 언제나 안전할 것입니다.
