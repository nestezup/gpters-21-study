# 플레이그라운드 완성 + 가상화 탐험

> **오늘의 목표**: 1~3주차에서 배운 도구들을 하나의 플레이그라운드로 연결하고, 가상화라는 기술을 탐험합니다.

---

## 나만의 플레이그라운드 만들기

3주 동안 코드(Git), 환경(fnm, WSL), 컨테이너(Docker) 세 가지 레이어에서 "되돌리기"를 배웠습니다. 이제 이걸 하나로 연결하면, 뭘 해도 되돌릴 수 있는 나만의 개발 연습장이 완성됩니다.

### 되돌리기 전체 지도

| 레이어 | 도구 | 되돌리기 방법 | 배운 주차 |
|--------|------|--------------|-----------|
| 코드 | Git | `git restore`, `git revert` → 커밋 단위로 복원 | 1주차 |
| 환경 | fnm | `fnm use 20` → 버전 전환 | 2주차 |
| 환경 | WSL | `wsl --unregister` → 삭제 후 재설치 | 2주차 |
| 컨테이너 | Docker | `docker rm` → 삭제 후 재생성 | 3주차 |
| OS 전체 | 가상머신 | 스냅샷 → 원하는 시점으로 복원 | 오늘! |

---

### 플레이그라운드 체크리스트

하나씩 체크하면서 내 환경을 점검해 봅시다.

**기본 도구 (2주차 복습)**
- [ ] 패키지 관리자 설치됨 (scoop 또는 brew) → `scoop --version` / `brew --version`
- [ ] Git 설치됨 → `git --version`
- [ ] fnm 설치됨 → `fnm --version`
- [ ] Node.js 설치됨 → `node --version`

**프로젝트 시작 (1주차 복습)**
- [ ] 폴더 생성 → `mkdir my-project && cd my-project`
- [ ] Git 초기화 → `git init`
- [ ] 첫 커밋 → `git add . && git commit -m "시작"`
- [ ] 코드 세이브 포인트 완성!

**Docker 준비 (3주차 복습)**
- [ ] Docker Desktop 실행 중 → `docker --version`
- [ ] 테스트 → `docker run hello-world`
- [ ] 서비스 하나 띄워보기 → 브라우저에서 확인
- [ ] 삭제 → 다시 띄우기 → 되돌리기 체험!

전부 체크했다면, 여러분은 이미 **"망가뜨려도 되돌릴 수 있는 사람"**입니다.

---

## 가상화 탐험 — 이런 게 있다!

여기서부터는 "반드시 해야 하는 실습"이 아니라, **이런 기술이 있다는 걸 알아두는 시간**입니다. 관심이 가면 직접 설치해보고, 아니면 구경만 해도 충분합니다.

### 가상화가 뭔가요?

내 컴퓨터 안에 소프트웨어로 된 독립 컴퓨터를 하나 더 만드는 기술입니다. 가상머신 안에서는 운영체제를 망가뜨려도 실제 컴퓨터에 영향이 없으므로, 마음껏 실험하고 스냅샷으로 되돌릴 수 있습니다.

Docker와의 차이: Docker는 앱 하나를 격리하고, 가상머신은 운영체제 전체를 격리합니다. 시스템 깊숙이 실습하려면 가상머신이 적합합니다.

### 어떤 도구를 쓸까?

| 내 환경 | 추천 도구 | 이유 |
|---------|----------|------|
| macOS (Apple Silicon / Intel) | **UTM** | 맥 전용, 무료, 오픈 소스 |
| Windows Pro / Enterprise / Education | **Hyper-V** | 윈도우 내장, 성능 우수 |
| Windows Home | **VMware** | Home에서도 사용 가능, 무료 |

> VMware는 Broadcom 인수 이후 라이선스 정책이 변경되었습니다. 설치 시점에 무료 사용 조건을 반드시 확인하세요.

---

### UTM (macOS)

#### 설치

```bash
brew install --cask utm
```

또는 공식 사이트(https://mac.getutm.app/)에서 .dmg 파일을 내려받아 Applications 폴더로 드래그합니다.

> 필수 조건: macOS 11 이상. Intel Mac에서도 동작합니다.

#### 가상머신 만들기

1. UTM 실행 → **새 가상머신 만들기** 클릭
2. **Virtualize** 선택 (내 하드웨어 성능을 최대한 활용)
3. **Linux** 선택 → 내려받은 Ubuntu ISO 연결
4. CPU 코어, 메모리(4GB 권장), 저장소(30~40GB) 설정
5. **재생 버튼** → 화면 안내에 따라 Ubuntu 설치 진행

#### 스냅샷

가상머신 정보 화면 → **스냅샷** 탭 → 현재 상태 저장

이름 예시: `실습 시작 전`, `패키지 설치 완료`

---

### VMware (Windows)

#### 설치

1. VMware 공식 다운로드 페이지에서 **Workstation Player** 설치 파일을 내려받습니다.
2. 설치 중 **비상업적 용도** 옵션을 선택하면 무료로 사용할 수 있습니다.
3. 설치 완료 후 재부팅이 필요할 수 있으니 작업 파일을 미리 저장하세요.

> 하드웨어 권장: RAM 8GB 이상. BIOS에서 CPU 가상화 기능이 켜져 있어야 합니다.

#### 가상머신 만들기

1. **새로 만들기** → Ubuntu ISO 선택
2. 이름, 저장 위치 지정
3. 하드웨어 설정: 메모리 4GB 이상, 디스크 30~40GB
4. 가상머신 시작 → Ubuntu 설치 진행

#### 스냅샷

상단 메뉴 → **스냅샷 찍기** → 이름 입력 후 저장

---

### Hyper-V (Windows Pro 이상)

#### 내 에디션 확인하기

```
Win + I → 시스템 → 정보 → Windows 사양
```

에디션이 **Pro, Enterprise, Education** 중 하나면 사용 가능합니다. Home이면 VMware를 사용하세요.

#### 설치 (기능 켜기)

**PowerShell (관리자 권한):**

```powershell
Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V -All
# 재부팅 후 시작 메뉴에서 "Hyper-V Manager" 검색
```

**GUI로 하기:**

제어판 → 프로그램 및 기능 → **Windows 기능 켜기/끄기** → Hyper-V 체크 → 확인 → 재부팅

#### 가상머신 만들기

1. Hyper-V Manager → 오른쪽 메뉴 → **새로 만들기 → 가상 컴퓨터**
2. 이름, 저장 위치 지정
3. 세대: **2세대** 권장 (최신 리눅스 호환)
4. 메모리: 4096MB(4GB) 이상
5. 네트워크: **Default Switch** (인터넷 자동 연결)
6. 가상 하드 디스크: 30~40GB
7. 설치 옵션: Ubuntu ISO 연결
8. **마침** → 가상머신 선택 → **시작**

#### 스냅샷 (체크포인트)

Hyper-V에서는 스냅샷을 **체크포인트**라고 부릅니다. 기능은 동일합니다.

가상머신 우클릭 → **체크포인트 만들기** → 되돌릴 때는 목록에서 선택 후 **적용**

#### 알아두어야 할 점

Hyper-V를 켜면 윈도우 자체가 하이퍼바이저 위에서 동작합니다. VMware, VirtualBox 등 다른 가상화 도구와 충돌할 수 있으니 함께 쓸 계획이라면 미리 고려하세요. 끄려면 같은 방법으로 체크박스 해제 후 재부팅합니다.

---

### Ubuntu ISO 내려받기

가상머신에 설치할 운영체제 이미지를 먼저 준비합니다. Ubuntu 공식 사이트에서 내 CPU 아키텍처에 맞는 ISO 파일을 내려받으세요.

- **Apple Silicon Mac** → ARM64 (aarch64) 버전
- **Intel Mac / Windows** → AMD64 (x86_64) 버전

---

### 일부러 망가뜨려 보기

가상머신을 설치했다면, 이 스터디의 정신을 제대로 체험해 봅시다.

**STEP 1:** 현재 정상 상태를 스냅샷(또는 체크포인트)으로 저장합니다.

**STEP 2:** 시스템을 고장 냅니다.

```bash
# 예시: 중요 시스템 파일 삭제 (가상머신 안에서만!)
sudo rm -rf /usr/bin
```

부팅이 안 되거나 화면이 깨져도 괜찮습니다.

**STEP 3:** 스냅샷/체크포인트 목록에서 아까 저장한 시점을 선택 → **복원/적용**

몇 초 만에 깨끗한 상태로 돌아옵니다. 이게 바로 "실수가 자유로운" 의 의미입니다.

## 삭제 및 정리

가상머신을 더 이상 쓰지 않는다면 정리합니다.

| 작업 | 방법 |
|------|------|
| 가상머신 삭제 | 목록에서 우클릭 → 삭제. **실제 파일까지 삭제**해야 저장 공간 회수 (수십 GB) |
| UTM 삭제 (Mac) | Applications 폴더에서 휴지통으로 이동 |
| VMware 삭제 (Win) | 제어판 → 프로그램 추가/제거 |
| 잔여 파일 확인 | 도구 삭제 후에도 가상머신 파일이 남아있을 수 있으니 저장 경로 확인 |

## 도구 비교

| 비교 항목 | UTM | VMware | Hyper-V |
| :--- | :--- | :--- | :--- |
| 추천 운영체제 | macOS | Windows (Home 포함) | Windows Pro 이상 |
| 비용 | 무료 | 개인 사용 시 무료 | 무료 (윈도우 내장) |
| 설치 방식 | 앱 설치 | 앱 설치 | 윈도우 기능 켜기 |
| 주요 강점 | 애플 실리콘 지원 최적화 | 높은 호환성과 안정성 | 윈도우 네이티브, 높은 성능 |
| 되돌리기 기능 | 스냅샷 | 스냅샷 | 체크포인트 |
| 주의 사항 | 맥 전용 | Broadcom 라이선스 확인 | 다른 가상화 도구와 충돌 가능 |

## 자주 만나는 문제

| 상황 | 원인 | 해결 |
|------|------|------|
| 가상머신이 너무 느림 | 메모리를 너무 많이/적게 할당 | 전체 메모리의 절반 이하로 조정 |
| 가상화 기능 비활성화 오류 | BIOS에서 가상화가 꺼져 있음 | 컴퓨터 부팅 시 BIOS 진입 후 가상화 옵션 활성화 |
| 화면 해상도가 작게 고정됨 | 게스트 확장 도구 미설치 | UTM/VMware 메뉴에서 게스트 확장 도구 설치 |
| 파일을 주고받고 싶음 | 공유 폴더 미설정 | 설정 메뉴에서 공유 폴더 경로 지정 |
| 인터넷 연결 안 됨 | 네트워크 설정 오류 | 네트워크 설정을 NAT 방식으로 변경 |
